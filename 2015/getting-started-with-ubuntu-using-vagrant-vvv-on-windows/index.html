Getting started with Ubuntu using Vagrant on Windows

Aug 6, 2015

1 minute read

Vagrant is a tool for building lightweight, reproducible, and portable development environment using configuration and provisioning script such as Varying Vagrant Vagrants. Although it is focused PHP and WordPress development, VVV also automatically provision major packages such as nginx, nodejs, and git.

There’re few things to take note if you’re keen to use VVV on Windows 8.1:

  1. VVV requires VirtualBox, thus you need to turn Hyper-V support off. Instead doing this manually, you can create new boot menu with a “No Hyper-V” entry using following tips by Scott Hanselman
  1. Hold down shift on the keyboard while clicking Windows Restart button
  2. Follow VVV step-by-step installation guide

  3. Update Windows host file with

     192.168.50.4 vvv.dev local.wordpress.dev local.wordpress-trunk.dev src.wordpress-develop.dev build.wordpress-develop.dev
  4. If you want to be able to use other devices (e.g. mobile phone) to access the virtual machine, you need to configure Vagrant with config.vm.network :public_network. Refer to webdevstudios’s post on How to View a Locally Developed Website on Other Devices.
  5. You can now use the Vagrant environment with vagrant up and vagrant halt
  6. Visit http://vvv.dev/ for accessing VVV default dashboard
  7. You may also need to use xip.io to access virtual hosts from devices on your local network, like iPads, iPhones.
